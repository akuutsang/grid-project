# 🧩 Admin Dashboard (The Odin Project)

This is a full admin dashboard layout project built using **HTML** and **CSS Grid**, as part of [The Odin Project](https://www.theodinproject.com/) Intermediate HTML & CSS course. The goal is to gain hands-on experience with **CSS Grid**, **layout structuring**, and **nesting containers**.

---

## 📌 Project Objective

Create a multi-section dashboard layout using **CSS Grid** as the primary layout tool. This includes a sidebar, header, and main content area filled with interactive UI elements like project cards, announcements, and trending content.

---

## 🛠️ Tools & Technologies

- HTML5
- CSS3
- CSS Grid (primary layout technique)
- Flexbox (where appropriate)
- Google Fonts (Roboto)
- Material Design Icons (SVG)

---

## 🧱 Project Setup & Structure

### Step 1: Set Up & Planning

- Initialized a Git repository.
- Created HTML and CSS files with dummy content to verify linkage.
- Reviewed the provided design layout to plan element structure.

### Step 2: Base Layout

- Created core layout containers:
  - `Sidebar`
  - `Header`
  - `Main Content`
- Applied **CSS Grid** to establish the base structure.

### Step 3: Nesting

- Used nested grid containers within each section:
  - Sidebar: Navigation and branding
  - Header: Search bar, user info, and buttons
  - Main Content: Projects, Announcements, Trending

### Step 4: Asset Integration

- Added dummy images and placeholder text for layout visualization.
- Downloaded icons from **Material Design Icons**.
- Applied **Roboto** font via Google Fonts.

---

## 💡 Development Tips Used

- Used colored borders and background shades for layout debugging.
- Mixed use of `fr` units and pixels for flexible track sizing.
- Practiced reusable grid patterns with nested grids.
- Focused on semantic HTML structure.

---

## 🎨 Customization & Design

- While inspired by the design example, this project is styled uniquely.
- Emphasis was placed on layout functionality and clean, structured code.
- This project is **not responsive by default**, but the project cards section was made scalable as a stretch goal.
